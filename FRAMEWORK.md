# 策略开发思维框架：认知 → 信息 → 行为 → 机制 → 策略

> 本文档总结了从周期趋势策略开发过程中提炼出的完整思维框架，适用于后续所有量化策略的设计与评估。

---

## 框架概览

```
认知限制（持久，无法被市场消除）
    +
信息结构（分散 vs 集中，决定偏差的传播方式）
    ↓
行为模式（集体行为，可以统计观测）
    ↓
市场机制（因果描述，描述性，不可直接使用）
    ↓  （需要：方向性 + 可观测信号 + 正期望值）
交易策略（规范性，可部署，有完整规则）
    ↓
寄生于特定周期阶段（无法逃离，只能选择寄生在哪个阶段）
```

---

## 第一层：认知（Cognition）

这是最底层，也是所有市场规律的起点。

人类大脑的结构性限制在市场中主要表现为：

| 认知限制 | 具体表现 |
|---------|---------|
| 注意力有限（有界理性） | 不能同时处理所有信息，只关注显著的变化 |
| 损失厌恶 | 对亏损的痛苦远大于对同等盈利的喜悦 |
| 锚定效应 | 价格被近期高低点锚定，影响决策 |
| 处置效应 | 过早卖出盈利，过久持有亏损 |
| 从众心理 | 在不确定情况下模仿他人行为 |

**关键结论**：认知限制是生理和神经系统层面的约束，无法通过市场机制消除，因此是持久的。这是行为 Alpha 不会随使用人数增加而快速衰减的根本原因。

---

## 第二层：信息（Information）

信息结构决定了认知偏差如何在市场中传播和放大。

### 两种信息结构，对应两类偏差来源

**分散型信息结构（多数市场的常态）**
- 信息在参与者之间逐渐扩散
- 偏差来源是个体认知错误的集合
- 产生的行为：羊群、FOMO、恐慌抛售
- 策略方向：趋势跟踪、均值回归

**集中型信息结构（政策主导、权力集中市场）**
- 关键信息掌握在少数人手里
- 偏差来源从认知层转移到结构层
- 产生的行为：全市场追踪信息源、政策驱动的过度反应
- 典型案例：A股政策周期、当前美股政策集中化

### 市场成熟度与偏差的关系

> 市场成熟度降低的是认知偏差的影响，但不能消除信息不对称偏差。

| 市场 | 主要偏差来源 |
|------|------------|
| A股 | 散户占比高（认知偏差为主）+ 政策集中（信息不对称叠加） |
| 美股（当前） | 算法、机构主导的认知偏差已弱化，但政策不确定性带来新的信息不对称 |
| 港股 | 夹在A股政策周期和全球资本之间，信息来源多元但方向冲突 |

**关键结论**：行为偏差不会因为市场成熟而消失，它只会换来源。越成熟的市场，结构性信息不对称的影响反而越显眼——因为认知偏差被对冲掉之后，剩下的主要就是信息层的不均衡。

## 第三层：行为（Behavior）

认知限制在特定信息环境下，产生可以观测的集体行为模式。

| 行为类型 | 认知根源 | 信息条件 | 市场表现 | 分类 |
|---------|---------|---------|---------|------|
| 羊群效应 / 恐慌 | 从众 + 损失厌恶 | 信息模糊、不确定 | 趋势延续、加速反转 | Type A |
| FOMO / 动量追逐 | 锚定 + 损失厌恶（错失） | 价格信号显著 | 追涨杀跌 | Type B |
| 过度反应 | 注意力偏向显著事件 | 单一事件冲击 | 短期过冲后回归 | Type C |
| 多周期盲点 | 注意力集中在单一时间框架 | 各时间维度信息不整合 | 短期噪声掩盖长期信号 | Type D |
| 注意力级联 | 注意力分配有时序 | 信息逐层扩散 | 量先于价格变化 | 机制层（非独立类型） |

**关键结论**：行为是认知限制和信息结构共同作用的结果，不能单独由一方解释。

---

## 第四层：机制（Mechanism）

行为模式反复发生，在市场价格序列中形成可以统计观测的规律。

### 机制的定义

机制是对市场现象的因果描述，解释"为什么会发生"。它是**描述性的**，不是操作性的。

### 内容不可预测 vs 过程可预测

| 层面 | 可预测性 | 说明 |
|------|---------|------|
| 内容层（触发事件） | 不可预测 | 下一个重大新闻、政策转向、黑天鹅 |
| 过程层（参与者反应） | 有规律 | 触发后，参与者如何反应，行为模式可观测 |

机制利用的是过程可预测性，不是内容可预测性。市场"不可预测"和"行为有规律"并不矛盾——它们在不同层级上各自成立。

### 机制与周期的关系

> 行为偏差是周期的产生机制，周期是行为偏差的宏观结果。

两者是同一现象的不同描述角度，不可分离。**所有利用机制的策略，都必然寄生于某个周期阶段。** 没有周期无关的策略，只有在不同周期阶段寄生的不同策略。

### 机制无法直接成为策略的条件

| 缺失条件 | 案例 |
|---------|------|
| 缺乏方向性 | 纯成交量信号：量大可能是买入也可能是恐慌卖出 |
| 信号代理质量差 | 宽基ETF成交量含申购赎回噪声 |
| 效果太小 | 扣除交易成本后 Alpha 消失 |
| 触发条件本身不可观测 | 需要区分是机构积累还是散户恐慌 |

> **重要区分**：注意力级联是一个已被记录的市场机制，但由于它缺乏方向性，无法独立构成策略，必须依附在方向判断层上才能形成完整策略。将机制误认为策略，是策略设计中最常见的错误之一。

## 第五层：策略（Strategy）

将机制转化为完整可部署系统的过程。

### 转化路径

```
Step 1：明确利用的偏差
  └─ 哪一类市场参与者？哪种认知限制？哪种信息结构？

Step 2：找到可观测的代理信号
  └─ 这个偏差在价格/量/时间序列上如何表现？

Step 3：确认方向性
  └─ 趋势 / 均值回归 / 套利？

Step 4：先定义失败条件（在什么市场状态下，这个偏差消失？）
  └─ 失败条件写不出来 → 对机制理解不足，暂停开发

Step 5：最小实现
  └─ 用最简单的方式表达机制，不要过早加复杂度

Step 6：独立验证（Walk-Forward，而非优化）
  └─ 验证的目标是找"在未来也成立的参数"，不是"历史最优参数"

Step 7：与同类策略对比
  └─ 是否真的在利用独立的偏差？是否比同类更简单的策略更好？
```

### 策略设计原则

**1. 一个策略对应一个偏差**

混合多个偏差会让失效归因变得不可能，也增加过拟合风险。当策略表现变差时，你无法判断是哪个偏差失效了。

**2. 简单优于复杂**

从11策略对比实验中，越基础的策略越稳健——因为简单策略的过拟合空间小，参数少，边界清晰。复杂策略可能在历史数据上看起来更好，但往往是在拟合噪声。

**3. 先写失败条件，再写成功条件**

知道策略在哪里不该用，比知道它在哪里好用更重要。失败条件是策略的边界，也是部署时最重要的风控信号。

**4. 偏差类型决定策略的周期依赖**

- Type A（羊群/恐慌）策略 → 寄生于趋势阶段
- Type B（FOMO/动量）策略 → 寄生于牛市加速段
- Type C（过度反应）策略 → 寄生于高波动事件后
- Type D（多周期盲点）策略 → 全周期更稳健，但信号频率低

选择策略等于选择寄生的周期阶段，没有例外。

## 三个核心问题（策略开发前必须回答）

在开始写第一行代码之前，先回答：

**Q1：我在利用哪一层的哪类偏差？**

是认知层（散户的系统性错误）还是信息层（信息传播不对称）？这决定了策略在什么市场结构下有效，以及 Alpha 的持久性。

**Q2：这个机制能独立转化为策略吗？**

它是否有方向性？代理信号质量如何？扣除成本后还有 Alpha 吗？触发条件是否可以直接观测？

**Q3：这个偏差在什么时候消失？**

先写失败条件。如果写不出来，说明对机制的理解还不够清楚，还不到写策略的时候。

## 本项目策略（v2）的框架定位

| 维度 | v2 策略的答案 |
|------|-------------|
| 利用的偏差 | Type A（羊群/恐慌）+ Type D（多周期盲点） |
| 信息结构假设 | 分散型，散户占主导 |
| 入场依据 | 趋势确立后（价格已在趋势中），利用羊群效应延续 |
| 方向性 | 明确：只做多，不做空 |
| 周期依赖 | 寄生于趋势阶段；震荡/快速轮动市场效果差 |
| 失败条件 | 高波动、快速轮动、无趋势资产（创业板 ETF 已验证） |
| Alpha 来源 | 行为 Alpha（纪律），非信息 Alpha |
| Alpha 持久性 | 高——因为行为偏差来自认知结构，不随策略使用人数增加而快速衰减 |
