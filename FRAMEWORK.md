# 策略开发完整框架（整合版）

> 整合行为金融理论（认知→机制路径）与海龟法则（执行系统），形成从"为什么有效"到"如何部署"的完整闭环。

---

## 框架的起点：Beta 与 Alpha 的分离

所有投资收益都可以分解为两部分：

```
总收益 = Beta 收益（承担系统性风险的补偿）+ Alpha（超额收益）
```

**Beta 收益是怎么来的**

Beta 不是策略的产物，而是市场存在的基础条件：

- 公司需要股权资本，投资者承担破产风险（股权优先级低于债权）
- 风险厌恶的投资者要求高于无风险利率的补偿 → 长期来看股票跑赢现金
- 经济增长 → 企业盈利增长 → 股价长期上涨
- 通胀补偿：股票是实物资产，长期名义回报包含通胀

**Beta 的行为金融补充（权益溢价之谜）**

理论模型预测的 Beta 远小于实际观测值。行为金融的解释：
- 投资者存在**短视损失厌恶**（Myopic Loss Aversion）：对短期亏损的恐惧，使他们要求远高于理性水平的风险溢价才肯持有股票
- 结果：权益资产长期被低估，Beta 收益比"应该"的更高

这意味着 **Beta 的量级本身也受行为偏差影响**，但 Beta 的存在是结构性的，不依赖行为偏差。

**Alpha 与 Beta 的关系**

本框架主要研究 Alpha——超越简单持有市场指数的超额部分。对于长多策略（如 v2），Alpha 的测量基准是同期持有不动的买入持有收益，而非零。

---

## 框架结构

这个框架有两个维度，缺一不可：

```
纵向（分析层）：解释 Alpha 的来源，定义策略的边界
横向（方向性）：理论和实践有两个相反的走向
```

**两个方向，描述同一件事：**

```
因果方向（理论）：机制来源 → 市场机制 → 策略设计 → 执行部署
发现方向（实践）：观察市场规律 → 推断机制来源 → 设计利用方式
```

两个方向都是合理的入口，最终需要在两端都能自洽：规律要有理论解释，理论要有经验验证。

## 分析层（六层结构）

### 层零：机制来源（Alpha 的四条产生路径）

市场中可利用的规律（机制）来自四个相互独立的来源：

```
[认知限制] × [信息结构] ──→ 行为模式 ──┐
                                          │
[基本面渐进修正]          ──────────────→┤
                                          ├──→ 机制 ──→ 策略 ──→ 执行
[制度性约束]              ──────────────→┤
                                          │
[市场微观结构]            ──────────────→┘
```

| 来源 | 产生机制 | Alpha 类型 | 主要适用场景 |
|------|---------|-----------|------------|
| **认知×信息（行为）** | 羊群效应、FOMO、恐慌延续 | 行为 Alpha（持久） | 散户占比高的市场 |
| **基本面渐进修正** | 盈利预期被分析师缓慢调整 | 信息 Alpha（中等） | 基本面有效传导的市场 |
| **制度性约束** | 指数调仓、机构规则性买卖 | 结构 Alpha（取决于规则） | 被动资金规模大的市场 |
| **市场微观结构** | 做市商对冲方向、流动性周期 | 结构 Alpha（取决于规则） | 机构主导、高频数据可得 |

本框架以**行为 Alpha 路径**为主，即第一条路径。其他三条路径适用于不同市场和策略类型，机制不同但分析逻辑相通：找到来源 → 定义信号 → 验证持久性。

---

### 层一+层二：认知（Cognition）× 信息（Information）

> 这两层是**并行输入关系**，共同产生行为，不是前后关系。

```
[认知限制] ──┐
              ├──→ 行为
[信息结构] ──┘
```

**认知层：持久的结构性约束**

| 认知限制 | 市场表现 |
|---------|---------|
| 注意力有限（有界理性） | 信息扩散有时序，早晚参与者行为不同步 |
| 损失厌恶 | 过早止盈、过久持亏；恐慌加速反转 |
| 从众心理 | 趋势自我强化，超过基本面支撑 |
| 近因偏差 | 追涨杀跌；熊市末期极度悲观 |
| 过度自信 | 仓位过重、止损过松；频繁交易 |
| 确认偏误 | 持亏时忽略负面信号，加重损失 |
| 锚定效应 | 整数关口、前期高低点产生支撑/阻力 |
| 处置效应 | 上涨阶段卖压提前出现 |

认知限制是神经系统层面的约束，无法通过市场机制消除，因此是**持久的**。

**信息层：塑造偏差的传播方式**

| 信息结构 | 偏差来源 | 典型市场 |
|---------|---------|---------|
| 分散型（多数市场） | 个体认知错误的集合 | A股散户主导阶段 |
| 集中型（政策/权力集中） | 信息持有者的决策行为 | A股政策窗口、当前美股 |

**反身性（Reflexivity）：价格本身是信息**

价格变化会成为新的信息，影响后续行为，形成自我强化的回路：

```
价格上涨 → 被视为趋势信号 → 更多买入 → 价格进一步上涨（超调）
```

反身性放大趋势的中段，也制造趋势末端的超调与随后的急速反转。趋势策略在反身性强的阶段效果最好；均值回归策略在反身性反转后效果最好。

---

### 层三：行为（Behavior）

认知 × 信息 → 可观测的集体行为模式。

| 类型 | 认知根源 | 市场表现 | 对应策略方向 |
|------|---------|---------|------------|
| **Type A** 羊群 / 恐慌 | 从众 + 损失厌恶 | 趋势延续、加速反转 | 趋势跟踪 |
| **Type B** FOMO / 动量 | 近因偏差 + 错失厌恶 | 追涨杀跌、牛市加速 | 动量策略 |
| **Type C** 过度反应 | 过度自信 + 注意力偏向 | 事件后短期过冲再回归 | 均值回归 |
| **Type D** 多周期盲点 | 注意力集中在单一时间框架 | 短期噪声掩盖长期信号 | 多周期框架 |

> **注**：注意力级联（量先于价格变化）不是独立的行为类型，而是 Type A/B 的时序扩散机制。它缺乏方向性，必须依附在方向判断层上才能形成策略。

---

### 层四：机制（Mechanism）

行为模式反复发生，形成统计可观测的市场规律。机制是**描述性的**，解释"为什么会发生"，但不等于策略。

**机制与周期的关系**

> 行为偏差是周期的产生机制；周期是行为偏差的宏观结果。两者描述同一件事。

| 市场周期阶段 | 主导行为 | 有效策略类型 |
|------------|---------|------------|
| 趋势上升段 | Type A（延续）+ Type B（FOMO）| 趋势跟踪 |
| 趋势下降段 | Type A（恐慌）+ Type C（过度反应）| 趋势跟踪（空）|
| 震荡段 | Type C（短期过冲）| 均值回归 |
| 周期转折 | Type D（盲点）+ 反身性反转 | 多周期框架 |

所有策略都寄生于某个周期阶段，没有例外。选择策略等于选择寄生的阶段。

**机制无法直接成为策略的条件**

| 缺失条件 | 案例 |
|---------|------|
| 缺乏方向性 | 纯成交量信号：量大可能是买入也可能是恐慌卖出 |
| 代理信号质量差 | 宽基ETF成交量含申购赎回噪声 |
| 效果被成本吃掉 | 信号频繁，手续费超过Alpha |
| 触发条件不可观测 | 无法区分机构积累与散户恐慌 |
| 信号已被定价 | 反身性加速了信息消化，信号失去领先性 |

### 层五：策略设计（Strategy Design）

**Alpha 类型分类（必须在设计前明确）**

| Alpha 类型 | 来源 | 持久性 | 衰减方式 |
|-----------|------|--------|---------|
| **行为 Alpha** | 认知偏差（损失厌恶、从众等） | 高 | 不随使用人数增加衰减，认知结构不变 |
| **信息 Alpha** | 信息不对称或更快的处理速度 | 低 | 使用者多 → 信息更快被定价 → Alpha 消失 |
| **结构 Alpha** | 市场规则、制度、产品结构 | 中 | 随监管变化而变化，可能突然消失 |

**策略设计转化路径**

```
Step 1：明确偏差类型与 Alpha 来源
  └─ 哪类参与者？哪种认知限制？行为/信息/结构 Alpha？

Step 2：找可观测的代理信号
  └─ 偏差在价格/量/时间序列上如何表现？代理信号质量如何？

Step 3：确认方向性
  └─ 趋势 / 均值回归 / 套利？缺乏方向性的机制不能独立成策略

Step 4：先定义失效条件（见下方失效类型）
  └─ 失效条件写不出来 → 对机制理解不足，暂停开发

Step 5：最小实现
  └─ 用最简单的方式表达机制；每增加一层，必须对应一个已定义的问题

Step 6：独立验证（Walk-Forward，而非优化）
  └─ 目标是找"在未来也成立的参数"，不是"历史最优参数"

Step 7：与同类策略对比
  └─ 是否利用了独立的偏差？比同类更简单的基准策略更好吗？
```

**失效类型分类（设计时必须明确）**

| 失效类型 | 触发原因 | 是否恢复 | 应对方式 |
|---------|---------|---------|---------|
| **周期性失效** | 当前市场阶段不适用 | 是，下一阶段恢复 | 暂停，等待阶段切换 |
| **机制衰减** | 偏差正被套利，同类策略增多 | 缓慢，不突然恢复 | 降低仓位，监控Alpha下降趋势 |
| **结构性失效** | 市场规则/产品结构改变 | 可能永久失效 | 重新评估机制假设是否成立 |

### 层六：执行系统（Execution System）

> 来自海龟法则的核心贡献：策略逻辑正确，但执行系统不完整，等于零。

策略层解决"做什么信号"，执行层解决"开多大仓、如何管理、怎么坚持"。

**仓位管理：基于波动率的单位制（ATR Unit）**

海龟用 ATR（真实波动幅度均值）而不是固定金额来定义每笔交易的风险单位：

```
1 Unit = 账户规模 × 单笔风险比例 / ATR(N)
```

这样保证每笔交易的实际风险是等量的，而不是名义金额相等。高波动期自动减仓，低波动期自动加仓。

**加仓规则（Pyramiding）**

趋势确认后，分批加仓而不是一次性建满：
```
初始建仓 1 Unit
每上涨 0.5×ATR 加 1 Unit
最多加至 4 Units（总仓）
```

每次加仓都要重新计算止损，确保整体风险不超限。

**止损规则：ATR 倍数止损**

```
止损位 = 入场价 - 2×ATR（多头）
```

止损基于市场自身的波动幅度，而非任意价格点位。

**仓位上限**

| 维度 | 海龟法则限制 | 原则 |
|------|------------|------|
| 单一标的 | 最多 4 Units | 避免单一资产集中风险 |
| 强相关标的合计 | 最多 8 Units | 相关资产视为同一风险源 |
| 方向合计（多/空） | 最多 12 Units | 避免方向性过度集中 |
| 全组合 | 最多 20 Units | 总风险上限 |

**策略组合：在偏差类型上分散**

同类偏差的策略在同一市场阶段会同时失效。分散化应该在**偏差类型**上分散：

```
Type A 策略 + Type C 策略 → 周期依赖不同，组合更稳健
Type A 策略 + 另一个 Type A 策略 → 表面两个策略，实质一个风险
```

---

### 纪律执行：理论理解是最好的纪律工具

海龟法则的核心难题不是规则本身，而是在连续亏损时还能继续执行。

**我们框架对执行纪律的贡献：**

如果你只知道规则（海龟法则路径），当策略回撤时，你无法判断是"应该继续"还是"规则失效了"，容易在底部放弃。

如果你理解机制（我们的框架路径），你可以判断：
- 当前市场处于震荡阶段（周期性失效）→ 这是预期内的回撤，继续执行
- Alpha 在连续多年的趋势阶段仍然下降（机制衰减）→ 这是结构性问题，需要评估

**理解"为什么有效"，是能在困难时期坚持执行的最重要原因。**

## 完整系统示意图

```
                    因果方向 →

[认知限制] ──┐                    策略    执行
              ├──→ 行为 ──→ 机制 ──→ 设计 ──→ 系统
[信息结构] ──┘                    ↑        ↑
                                  发现方向  海龟贡献

                    ← 发现方向

反馈①（反身性）：价格 → 信息 → 行为 → 价格（自我强化）
反馈②（策略冲击）：策略大量使用 → 机制削弱 → Alpha衰减
```

## 三个核心问题（策略开发前必须回答）

**Q1：Alpha 从哪里来，持久性如何？**
是行为 Alpha（认知偏差，持久）、信息 Alpha（信息优势，衰减快）、还是结构 Alpha（制度红利，取决于规则）？

**Q2：这个机制能独立转化为策略吗？**
有方向性吗？代理信号质量如何？扣除成本后还有 Alpha 吗？信号是否已被反身性定价进去？

**Q3：失效时是暂时性还是结构性？**
周期性失效（会恢复） → 暂停策略。机制衰减（缓慢下降） → 降仓监控。结构性失效（规则改变）→ 重新评估假设。

---

## 监控原则（部署后）

| 监控指标 | 正常回撤 | 机制失效信号 |
|---------|---------|------------|
| 回撤时间 | 与历史最大回撤周期相近 | 显著超过历史 |
| 回撤与周期匹配度 | 在不适用阶段回撤 | 在适用阶段仍然亏损 |
| Alpha 年度趋势 | 在均值附近波动 | 连续多年单方向下降 |
| 同类策略对比 | 同偏差类型策略也回撤 | 仅本策略失效，同类仍有效 |

## 本项目 v2 策略的完整定位

| 维度 | v2 策略 |
|------|--------|
| 利用的偏差 | Type A（羊群）+ Type D（多周期盲点） |
| Alpha 类型 | 行为 Alpha（高持久性） |
| 信息结构假设 | 分散型，散户认知偏差驱动 |
| 反身性作用 | 趋势中段被反身性放大，信号有效期更长 |
| 执行层（缺口） | 目前无 ATR 单位制；加仓规则待补充 |
| 周期性失效 | 震荡/快速轮动市场（已在创业板 ETF 中验证） |
| 结构性失效条件 | A股散户占比大幅下降；日线信号被高频算法压缩 |
| 仓位建议（待完善） | 参考 ATR 单位制，按 1-2% 单笔风险计算头寸 |

## 两个框架的互补关系

| 维度 | 行为金融框架（本文） | 海龟法则 |
|------|-------------------|---------|
| 解释 Alpha 来源 | 完整 | 缺失（隐含假设） |
| 失效条件定义 | 完整 | 缺失（只接受回撤） |
| 策略设计逻辑 | 完整 | 经验归纳 |
| 仓位管理 | 缺失 | 完整（ATR单位制） |
| 加仓与止损规则 | 缺失 | 完整 |
| 纪律执行支撑 | 理论理解 | 规则纪律 |
| 适应市场变化 | 有判断框架 | 规则不变 |

**结论：两者相加才是完整的策略系统。** 行为金融框架告诉你为什么做、边界在哪里、什么时候停；海龟法则告诉你做多大、怎么加仓、如何控制损失。
